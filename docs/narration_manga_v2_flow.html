1
分镜剧本生成 LLM
stage_storyboard
输入

小说文本（≤15,000 字）

输出

storyboard.json

处理流程
  • LLM 阅读小说文本，识别"有效冲突单元"
  • 每个 unit 包含：title / core_conflict / emotion_tone / key_scenes / script
  • script 是 narration + action 列表（无 dialogue 类型）
  • narration: 旁白文字 (20-30字)，推演剧情，绝不描述画面
  • action: 画面描写，描述角色动作和场景变化
  • 同时输出 character_profiles（无 voice_trait 字段）
  • 校验：_validated_chat_json() 检查 units + character_profiles，最多重试 3 次

1472 × 832 px · 白色背景 · 三视图
(正面+侧面+背面) · 动漫风格

输出

characters/charref_{cid}_v{n}.png

注册到 CandidateManager: char_ref:{cid}

4
4×4 宫格分镜图 LLM Gemini 4K
stage_storyboard_grids
Step A: LLM 生成 16 shots
  • 每个 shot: 英文关键词 prompt (20-30 词)
  • Tags 格式: [景别] + [主体动作] + [环境] + [风格]
  • 每个面板为 9:16 竖屏构图
  • 节奏规则：每 3-4 镜头景别跳变
  • _strip_shot_labels() 去除景别缩写
Step B: Gemini 生成宫格图
  • 4K 输出: 9:16 portrait 整体
  • 参考图: 角色三视图 + 场景参考图
  • Prompt: 180° 规则 / 30° 规则 / 节奏规则
  • 输出: grids/grid_u{n}_v{n}.png
5
切图 + 视频分镜指令 PIL LLM
stage_video_prompts
Step A: PIL 4×4 切图
  • _detect_grid_panels() 自动检测面板边界
  • 16 帧 resize 到统一尺寸 (portrait)
  • 注册到 CandidateManager
  • frames/u{n}/frame_{01-16}.png
Step B: LLM 生成 16 段指令（旁白版）
  • 每帧对应一段 (16 帧 → 16 段)
  • 无 is_dialogue / dialogue / facing 字段
  • 有 narration_text / is_memory / emotion
  • video_prompt 禁止说话/口型描写
  • 时长: 旁白=字数÷4s, 动作=3-5s, 最小 3s
  • 输出: video_segments_u{n}.json
6
旁白 TTS + 时长校准 LLM Qwen TTS
stage_narration_tts
Step 1: LLM 选择旁白音色
  • 输入：剧本标题 / 情感基调 / 核心冲突 / 角色 / 旁白摘要
  • LLM 从 Qwen VOICES 音色库中选择最佳音色
  • 同时生成 story-aware TTS instruct (50-80字)
  • instruct 描述声音质感 + 节奏变化 + 情感层次
  • 保存至 narration_voice.json
Step 2: 逐段 TTS 生成
  • 仅处理 narration_text 非空的段
  • 根据段落情感微调 instructions
  • 调用 qwen_tts(text, voice, instructions)
  • 输出 .wav 格式
  • 回写 tts_path / tts_duration / final_duration 到 JSON
时长校准公式

final_duration = max(estimated, ⌈tts_duration⌉, 3)

  • 无 +1s buffer — 精确对齐
  • 非旁白段: final = max(estimated, 3)
与 dialogue_manga 的关键区别
dialogue_manga 使用 MiniMax TTS (21 中文音色) 为每个角色独立生成配音，需要维护 voice_map.json 和 voice_library.json。
narration_manga_v2 使用 Qwen3-TTS-Instruct-Flash 单旁白 TTS，LLM 自动选择音色并生成与故事匹配的 instruct 指令。无 voice_library，无回忆混响。
7
视频生成 Kling V3
stage_video_gen
Plan C: 首帧 Only + Serial
  • 只用 start_frame，无 image_tail
  • 串行执行：每段完成后等 5s 再继续
  • 硬切过渡，不加 crossfade
  • Kling V3 image2video + subject_reference
9:16 + sound=on
  • aspect_ratio: "9:16" 竖屏
  • sound: "on" Kling 自动生成环境音效
  • 环境音效作为三层混合的 Layer 1
  • 无 lip-sync（旁白在 assembly 叠加）
参数配置

modestd
aspect_ratio9:16
soundon

subject_reference角色三视图 (最多 1 张)
8
旁白字幕压制 FFmpeg
stage_subtitle_burn
字幕样式
  • 旁白: █ 白色, 28px, 黑色描边 2px
  • 位置: 底部居中 y=h-th-60
  • 字体: _resolve_cjk_font() 自动检测
  • 仅旁白段（narration_text 非空）有字幕
自动折行规则
  • 每行最大 20 个中文字
  • 竖屏 720px 宽 - 边距 40px×2 = 640px
  • 优先在标点处断句（，。！？；、）
  • 标点必须在行中间以后才断
长文本分段逻辑
  • 折行后 ≤2 行：一次性显示全段时长
  • 折行后 >2 行：拆成 2 段，各显示一半时长
  • 使用 enable='between(t,start,end)' 控制分段显示
  • 多段字幕通过逗号连接的多个 drawtext filter 叠加

她望着远处的灯火，
心中涌起一阵孤独。
9
组装 + 三层音频混合 FFmpeg ElevenLabs
stage_assembly
Step A: 收集 & 帧率统一
  • 字幕版优先，无字幕则用原版
  • 全部 clip → 720×1280 @30fps
  • 无音频的 clip 添加静音轨
  • libx264, preset=fast, crf=18
Step B: 拼接
  • FFmpeg concat demuxer
  • 硬切，无过渡效果
  • concat_u{n}.txt → u{n}_concat.mp4
Step C: 旁白时间轴音轨
  • 静音基底 anullsrc (duration=total)
  • 各段 TTS 按 adelay 偏移叠加
  • amix 合成完整旁白音轨
  • 输出: u{n}_narration_track.mp3
Step D: BGM
  • LLM 生成音乐 prompt (50-80 词英文)
  • 要求纯器乐 (instrumental only, 禁止人声)
  • ElevenLabs Music API 生成 BGM
  • BGM 预处理 → u{n}_bgm_processed.mp3
Step E: 三层音频混合
Layer 1: 视频环境音
-35 dB
Layer 2: 旁白 TTS
mean -15 dB
Layer 3: BGM 器乐
mean -28 dB
  • Video audio: 固定 volume=-35dB (环境底噪)
  • Narration: 动态校准 adjust = -15 - measured_mean, 防 clipping < -1dB
  • BGM: 动态校准 adjust = -28 - measured_mean, fade out 1s + 防 clipping
  • amix: inputs=N, duration=first, dropout_transition=2
  • 输出编码: AAC 192kbps
BGM 预处理 (compand + dynaudnorm)
compand=attacks=0.1:decays=0.5:points=-90/-90|-45/-20|-30/-16|-20/-16|0/0:gain=0
dynaudnorm=framelen=3000:gausssize=31:peak=0.95:maxgain=25:correctdc=1
  • Step 1: compand 向上压缩 — -30dB 以下提升最大 +15dB，-20dB 以上不动
  • Step 2: dynaudnorm 滑窗均衡 — 3s 窗口 (framelen=3000)，maxgain=25dB，peak=0.95
  • 效果：避免 ElevenLabs BGM 首尾渐入渐出段音量过小
10
质量检测 Quality Gate
stage_quality_gate
检测项目
检测项判定条件级别
视频时长 duration < target × 0.5 FAIL
BGM 可听性 |concat_mean - output_mean| < 1.0 dB FAIL
视频完整性 generated_segments < total_segments FAIL
旁白覆盖率 tts_with_file < narration_segments FAIL
通过

所有检测项均通过 → quality_passed = True

失败

任一检测项失败 → 记录 issues 列表
不阻断流程，仅标记 + 日志输出

抽卡 / REROLL
🎲
Reroll 抽卡体系
op_reroll_*
目标方法策略版本上限
宫格帧 op_reroll_frame FIFO 切换备选 → 重生成整张宫格 → 切 16 帧，目标帧自动选中 3
视频段 op_reroll_video_segment 重新 Kling V3 image2video (9:16, sound=on)，参数同原始生成 无限制
旁白 TTS op_reroll_narration_tts 重新 TTS 生成，支持 voice_id 覆写，回写 final_duration 无限制
角色参考图 op_reroll_char_ref Jimeng T2I 重新生成三视图（继承 InteractiveOpsMixin） 无限制
场景背景 op_reroll_scene_bg Gemini 重新生成场景参考图（继承 InteractiveOpsMixin） 无限制
V2 专用覆写
op_reroll_tts() 和 op_reroll_video() 被覆写为错误提示，引导使用 V2 专用方法。
op_reroll_narration_tts 需要 --unit 和 --seg 参数。
op_reroll_video_segment 需要 --unit 和 --seg 参数。

架构
🏗
系统架构
工作流引擎
  • BaseWorkflow — stage 执行框架
  • InteractiveOpsMixin — 通用交互操作
  • @register_workflow — 模板注册
  • CandidateManager — 资产版本管理
接入方式
  • CLI: scripts/e2e_v11b.py
  • REST API: app/api/v1/workflows.py
  • Python: get_workflow("narration_manga_v2")
断点恢复

每个 stage 检查已有输出文件，跳过已完成的步骤。CandidateManager 跟踪所有资产版本和选中状态。stop_after_stage 支持分段执行。

LLM 校验

_validated_chat_json() 统一校验 LLM 输出：检查必需字段 + 列表字段 + 列表长度，最多重试 3 次，失败抛出 RuntimeError。

共享工具函数 (_shared.py)
  • _resolve_cjk_font — 自动检测 CJK 字体
  • _ffmpeg_safe_text — 转义 FFmpeg drawtext 特殊字符
  • _strip_shot_labels — 去除景别缩写标签
  • _detect_grid_panels — 自动检测宫格面板边界
  • _preprocess_bgm — BGM 音量均衡预处理
  • _measure_mean_volume — 测量音频均值音量
