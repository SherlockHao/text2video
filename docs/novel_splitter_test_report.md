```

           │

           │
           ▼

│ Step 3: extract_episodes    │  基于锚点从原文精准截取

           │

```

**函数**: `split_into_chunks(text, max_length=5000)`

| 指标 | 值 |
|---|---|
| 输入字数 | 630 |
| max_length | 5000 |
| 输出 chunk 数 | 1 |

## 4. Step 2 — LLM 规划师调用

### 4.1 发送给 LLM 的 Prompt

**System Prompt**:
```text
# Role: 顶级短剧总编剧兼全剧架构师 (Showrunner)

# Task:
阅读提供的【小说文本块】，将其大刀阔斧地"脱水提纯"，拆解为 1 集左右的高节奏、
强冲突短剧/漫剧章节，并输出一份用于工程切割的《剧集拆解蓝图》(JSON格式)。

# Core Rules:
1. 剧情脱水：毫不留情地砍掉冗长景物描写和无关支线。每集只保留核心冲突。
2. 悬念钩子：每集的结尾必须卡在情绪最高潮或冲突即将爆发的瞬间。
3. 状态记忆：为每一集生成精准的 context_state。
4. 物理切割锚点：从原文中提取原封不动的句子作为 start_anchor 和 end_anchor。

# Output Format:
严格遵守且仅输出JSON格式。
```

**User Prompt**:
```text
【前情提要】：无
(注：如果前情提要为"无"，说明这是小说的开头。)

【目标集数】：请尽量将以下文本块拆解为 1 集。

【小说文本块】：
{完整 630 字原文}
```

**参数**:
| 参数 | 值 |
|---|---|
| model | qwen3.5-plus |
| temperature | 0.5 |
| max_tokens | 4096 |
| target_episodes | 1 (= 630 // 600) |
| previous_context | "无" (首个 chunk) |

## 7. 下游对接说明

每集的 `ep_XXX.txt` 可以直接作为现有分镜工作流 (`narration_manga`) 的 `input_text` 输入：

```bash
# 以第1集为输入，运行分镜→视频全流程
python scripts/e2e_v11b.py run \
  --input e2e_output/split_test/episodes/ep_001.txt \
  --output e2e_output/ep_001 \
  --duration 60
```

分镜 LLM 可以额外利用 `ep_XXX_meta.json` 中的：
- **`context_state`** — 作为前情提要注入分镜 prompt，防止角色/场景失忆
- **`compression_instruction`** — 作为指导，告诉分镜模型该强调什么、删掉什么

## 9. 三级锚点容错机制

| 级别 | 方式 | 触发条件 |
|---|---|---|
| **exact** | `text.find(anchor)` 精确匹配 | 锚点与原文一字不差 |
| **fuzzy** | 取前8字子串 / 去标点滑动匹配 | LLM 多加/少加标点 |
| **fallback** | 用上一集 end / 下一集 start / chunk 末尾 | 锚点完全找不到 |
