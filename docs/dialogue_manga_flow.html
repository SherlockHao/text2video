对话漫剧工作流

dialogue_manga — 多角色对话驱动的漫剧推广视频生成 · 10 Stage Pipeline

Agent 团队
8 个 AI 服务协作
Qwen 3.5-plus — LLM Jimeng T2I — 角色图 Gemini 3.1 Flash — 场景/宫格 Kling V3 — 视频生成 Kling V3 — Lip-sync MiniMax — TTS ElevenLabs — BGM FFmpeg + PIL — 后处理
1
分镜剧本生成 LLM
stage_storyboard
输入

小说文本（≤15,000 字）

输出

storyboard.json

处理流程
  • LLM 阅读小说文本，识别"有效冲突单元"
  • 每个 unit 包含：title / core_conflict / emotion_tone / key_scenes / script
  • script 是 dialogue + action 列表，每条标注说话角色
  • 同时输出 character_profiles：char_id / appearance_prompt / voice_trait
  • 校验：_validated_chat_json() 检查 units + character_profiles 字段，最多重试 3 次
2
角色三视图参考图 Jimeng T2I
stage_char_refs
规格

1472 × 832 px · 白色背景 · 三视图
(正面+侧面+背面) · 动漫风格

输出

characters/charref_{cid}_v{n}.png

注册到 CandidateManager: char_ref:{cid}

Prompt 构成

杰作 + 动漫风格 + 角色设定图 + 白色背景 + 三视图 + gender_hint + appearance_prompt + 全身立绘 + 无文字无水印

3
角色音色库 LLM MiniMax FFmpeg
stage_char_voices
Step 1: LLM 音色匹配
  • 输入：角色 voice_trait + 21 种 MiniMax 音色描述
  • 输出：voice_id / emotion / intro_text
  • 保存至 voice_map.json
Step 2: TTS 生成
  • MiniMax TTS 生成自我介绍台词 (speed=0.9)
  • 主音色：voice_{cid}_v{n}.mp3
  • 回忆版：FFmpeg aecho 混响 → voice_{cid}_memory_v{n}.mp3
回忆混响参数

aecho=0.8:0.7:40|60:0.3|0.2, highpass=f=80, lowpass=f=6000

4
场景参考图 Gemini
stage_scene_refs
特点

带角色参考图输入，保持角色一致性

基于 key_scenes 的 location + description

输出

scenes/scene_u{n}_s{n}_v{n}.png

注册: scene_ref:u{n}_s{n}

参考图传递

将本 unit 涉及的角色三视图作为 Gemini 的 ref_images 输入，附带角色名和外貌标签

5
4×4 宫格分镜图 LLM Gemini 4K
stage_storyboard_grids
Step A: LLM 生成 16 shots
  • 每个 shot: 英文关键词 prompt (20-30 词)
  • Tags 格式: [景别] + [主体动作] + [环境] + [风格]
  • 节奏规则：每 3-4 镜头景别跳变
  • _strip_shot_labels() 去除景别缩写避免渲染文字
Step B: Gemini 生成宫格图
  • 4K 输出: 5504 × 3072 (16:9)
  • 参考图: 角色三视图 + 场景参考图
  • Prompt 包含: 180° 规则 / 30° 规则 / 节奏规则
  • 输出: grids/grid_u{n}_v{n}.png
6
切图 + 视频分镜指令 PIL LLM
stage_video_prompts
Step A: PIL 4×4 切图
  • 5504×3072 ÷ 4 = 每格 1376×768
  • 16 帧注册到 CandidateManager
  • frames/u{n}/frame_{01-16}.png
Step B: LLM 生成 15 段指令
  • 相邻两帧组成一段 (帧N → 帧N+1)
  • 每段: is_dialogue / is_memory / dialogue / facing
  • 时长估算: 台词=字数÷4s, 动作=3-5s
  • 输出: video_segments_u{n}.json
7
对话 TTS + 时长校准 MiniMax FFmpeg
stage_dialogue_tts
处理流程
  • 仅处理 is_dialogue=true 的段
  • 按 char_id 查询 voice_map 获取 voice_id + emotion
  • 去除括号注释后调用 MiniMax TTS
  • 回忆场景: 额外 FFmpeg aecho 加混响
时长校准公式

final_duration = max(estimated, ⌈tts_duration⌉, 3)

  • 无 +1s buffer — 精确对齐
  • 非台词段: final = max(estimated, 3)
  • 回写 tts_path / tts_duration / final_duration 到 JSON
8
视频生成 Kling V3
stage_video_gen
Plan C: 首帧 Only
  • 只用 start_frame，无 image_tail
  • 全部段可并行生成，无累积误差
  • 硬切过渡，不加 crossfade
  • Kling V3 image2video + subject_reference
Lip-sync 流程
  • image2video 完成后获取 video_id
  • POST /v1/videos/lip-sync
  • audio2video 模式, audio_file = base64(mp3)
  • 轮询等待完成 (最长 600s)
  • 失败时降级为原始视频
Lip-sync 决策树
is_dialogue ? → 否: 直接下载原始视频
→ 是: is_memory ? → 是: 跳过 lip-sync (回忆场景)
→ 否: "背" in facing ? → 是: 跳过 lip-sync (背对镜头)
→ 否: 执行 lip-sync
9
字幕压制 FFmpeg
stage_subtitle_burn
字幕样式
  • 角色名: ■ #FFD700 金色, 22px, 黑色描边 1.5px
  • 台词: ■ 白色, 28px, 黑色描边 2px
  • 位置: 底部居中 (y=h-th-60 / y=h-th-95)
  • 字体: PingFang SC (macOS 内置)
处理范围
  • 仅 is_dialogue=true 的段
  • 去除括号注释后显示
  • 非台词段保持原视频不变
  • 输出: u{n}_seg{nn}_subtitled.mp4
10
组装输出 FFmpeg ElevenLabs
stage_assembly
Step A: 帧率统一
  • 全部 clip → 1280×720 @30fps
  • Kling image2video 输出 24fps
  • Kling lip-sync 输出 30fps
  • 无音频的 clip 添加静音轨
Step B: 拼接
  • 字幕版优先，无字幕则用原版
  • FFmpeg concat demuxer
  • 硬切，无过渡效果
Step C: BGM
  • LLM 生成音乐 prompt (50-80 词英文)
  • ElevenLabs Music API 生成 BGM
  • BGM volume=0.15, 淡入 2s, 淡出 2s
  • amix 混合：原始音频 + BGM
最终输出

videos/u{n}_output.mp4

1280×720 · 30fps · AAC 128k

抽卡 / REROLL
🎲
Reroll 抽卡体系
op_reroll_*
目标方法策略版本上限
宫格帧 op_reroll_frame FIFO 切换备选 → 重生成整张宫格 → 切 16 帧，目标帧自动选中，其他帧作为备选 3
视频段 op_reroll_video_segment 重新 Kling V3 image2video + lip-sync 全流程 无限制
对话 TTS op_reroll_dialogue_tts 重新 MiniMax TTS，支持 voice_id 覆写，回忆场景自动加混响 无限制
角色参考图 op_reroll_char_ref Jimeng T2I 重新生成三视图 无限制
场景背景 op_reroll_scene_bg 重新生成场景参考图 无限制
审查 / REVIEW
🔍
Review 审查操作
op_review_*
操作返回内容
review storyboard所有 units 摘要 (title/conflict/emotion/characters/dialogue_count) + character_profiles
review status10 个 stage 完成状态 + 各类资产数量统计
review characters角色三视图路径 + 音色信息 (voice_id/emotion/samples)
review unit {n}指定 unit 的 15 段分镜详情 (帧路径/视频路径/TTS路径/时长)
review assets {type}指定类型的全部资产列表 (scene_refs/grids/frames/videos)
架构
🏗️
系统架构
工作流引擎
  • BaseWorkflow — stage 执行框架
  • InteractiveOpsMixin — 通用交互操作
  • @register_workflow — 模板注册
  • CandidateManager — 资产版本管理
接入方式
  • CLI: scripts/e2e_v11b.py
  • REST API: app/api/v1/workflows.py
  • Python: get_workflow("dialogue_manga")
断点恢复

每个 stage 检查已有输出文件，跳过已完成的步骤。CandidateManager 跟踪所有资产版本和选中状态。stop_after_stage 支持分段执行。

LLM 校验

_validated_chat_json() 统一校验 LLM 输出：检查必需字段 + 列表字段 + 列表长度，最多重试 3 次，失败抛出 RuntimeError。
