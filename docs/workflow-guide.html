1. 整体流程概览

输入文本 ──→ [Stage 1] LLM 分镜 ↓ ← 可暂停：review / edit 分镜 [Stage 2] TTS 语音 ↓ ← 可暂停：试听 / reroll TTS [Stage 3] 时长规划（自动，基于真实 TTS 时长） ↓ [Stage 4] 角色参考图 ↓ ← 可暂停：预览 / 抽卡 / 选择 [Stage 5] 场景背景图 ↓ ← 可暂停：预览 / 抽卡 / 选择 [Stage 6] 首帧图 ↓ ← 可暂停：预览 / 抽卡 / 选择 [Stage 7] 视频生成 ↓ ← 可暂停：预览 / 抽卡 / 选择 [Stage 8] 组装（字幕 + BGM + 拼接） ↓ [Stage 9] 质量检测 ↓ ★ final_video.mp4
核心理念：run 是批量自动执行（支持断点恢复）， review/edit/reroll/select 是独立的交互操作。 Agent 或 UI 通过 run --stop-after-stage xxx 暂停 → 执行交互操作 → 再 run 继续。

2. 快速开始（3 分钟跑通）

一键跑完全流程

# 从输入文本直接生成最终视频
python scripts/e2e_v11b.py run \
  --output e2e_output/my_video \
  --duration 40 \
  --input data/test_novel.txt

断点恢复（崩溃后重跑）

# 同一个 output 目录再跑一次，自动跳过已完成的部分
python scripts/e2e_v11b.py run --output e2e_output/my_video

查看状态

python scripts/e2e_v11b.py review status --output e2e_output/my_video --json

3. 九阶段详细流程

1 storyboard — LLM 分镜

调用 Qwen 3.5-Plus，将输入文本拆解为 4-5 个叙事段（segment），每段包含旁白 + 2-3 个子镜头。

输出：storyboard.json

断点恢复：如果 storyboard.json 已存在，直接加载。

2 tts — 语音合成

调用 MiniMax TTS，为每个叙事段的旁白文本生成带情感的语音音频。

输出：audio/seg_01_v1.mp3 ...

断点恢复：已有音频的段跳过，只补生成缺失的。

3 duration_plan — 时长规划

基于真实 TTS 时长（而非估算），自动调整每个子镜头的视频时长。

4 种情况：

Case条件处理
1TTS+1s ≥ 子镜头总时长保持不变
2TTS+1s < 总时长，缩短后均≥3s等比例缩短
3缩短后有<3s钉死3s，缩短其他
4全部3s仍超出LLM重新生成为单镜头
4 char_refs — 角色参考图

调用即梦 T2I，为每个角色生成动漫风格参考图（832×1472）。

用途：作为 Kling V3 的 subject_reference，保证角色跨镜头一致。

支持抽卡：可以 reroll 生成多个候选，选择最满意的。

5 scene_bgs — 场景背景图

每个场景一张无人物的背景图，仅供审查。

6 first_frames — 首帧图

场景首段第一个子镜头用 T2I 生成，其他子镜头用前一个视频的末尾帧。

7 video_gen — 视频生成

调用 Kling V3 I2V，每个子镜头生成 3-5 秒视频（动态时长）。

输入：首帧图 + 角色参考图 + motion prompt

支持抽卡：不满意可以 reroll。

8 assembly — 组装

FFmpeg：子镜头拼接 → 与 TTS 对齐 → 所有段拼接 → 字幕 → BGM

9 quality_gate — 质量检测

检查总时长、BGM、视频完整性、场景连续性。

4. 交互操作详解

4.1 Review（审查）

查看当前各环节的状态和输出。

# 查看整体状态
python scripts/e2e_v11b.py review status --output DIR --json

# 查看分镜脚本
python scripts/e2e_v11b.py review storyboard --output DIR --json

# 查看 TTS 音频状态（含时长）
python scripts/e2e_v11b.py review tts --output DIR --json

# 查看各类图片/视频资产
python scripts/e2e_v11b.py review char_refs --output DIR --json
python scripts/e2e_v11b.py review scene_bgs --output DIR --json
python scripts/e2e_v11b.py review videos --output DIR --json

4.2 Edit（编辑分镜）

修改分镜脚本中的字段，自动级联失效下游资产。

# 修改某段的旁白文本（自动失效 TTS + 视频）
python scripts/e2e_v11b.py edit storyboard \
  --output DIR --segment 2 \
  --field narration_text --value "新的旁白文字"

# 修改某段的情感标签（自动失效 TTS）
python scripts/e2e_v11b.py edit storyboard \
  --output DIR --segment 1 \
  --field emotion --value angry

# 修改某个子镜头的动作描述（自动失效对应视频）
python scripts/e2e_v11b.py edit storyboard \
  --output DIR --segment 3 --sub 2 \
  --field video_prompt --value "女子缓缓转头，泪水滑落"
注意：编辑后，下游资产会被标记为"失效"。下次执行 run 时会自动重新生成失效的部分。

4.3 Reroll（抽卡）

对不满意的资产生成新的候选项。旧的候选项保留，可以在多个候选中选择。

# 角色参考图抽卡
python scripts/e2e_v11b.py reroll char_ref --output DIR --char char_001

# 场景背景图抽卡
python scripts/e2e_v11b.py reroll scene_bg --output DIR --scene scene_001

# TTS 重新生成（可指定不同情感）
python scripts/e2e_v11b.py reroll tts --output DIR --seg 2 --emotion fearful

# 视频抽卡（seg=段号, sub=子镜头号，1-based）
python scripts/e2e_v11b.py reroll video --output DIR --seg 1 --sub 2

4.4 Select（选择候选）

在多个候选项中选择最终使用的版本。

# 先查看所有候选项
python scripts/e2e_v11b.py list-candidates char_ref \
  --output DIR --char char_001 --json

# 输出示例:
# {
#   "asset_key": "char_ref:char_001",
#   "candidates": [
#     {"version": 1, "path": "...", "is_selected": true},
#     {"version": 2, "path": "...", "is_selected": false},
#     {"version": 3, "path": "...", "is_selected": false}
#   ]
# }

# 选择第 2 个候选
python scripts/e2e_v11b.py select char_ref \
  --output DIR --char char_001 --candidate 2

# 选择视频候选
python scripts/e2e_v11b.py select video \
  --output DIR --seg 1 --sub 1 --candidate 2

5. Agent 完整工作流示例

以下是 AI Agent 通过 CLI 完成一个高质量视频的完整交互流程：

1 生成分镜脚本
python scripts/e2e_v11b.py run \
  --output e2e_output/project_001 \
  --duration 40 \
  --stop-after-stage storyboard
2 Agent 审查分镜
python scripts/e2e_v11b.py review storyboard \
  --output e2e_output/project_001 --json

Agent 分析 JSON 输出，判断旁白质量。

3 Agent 修改不满意的旁白
# 改第 2 段旁白
python scripts/e2e_v11b.py edit storyboard \
  --output e2e_output/project_001 \
  --segment 2 --field narration_text \
  --value "推开门的一瞬间，她僵住了——那个消失三年的男人，正冷冷看着她。"

# 改第 3 段情感
python scripts/e2e_v11b.py edit storyboard \
  --output e2e_output/project_001 \
  --segment 3 --field emotion --value whisper
4 继续执行到角色图生成完毕
python scripts/e2e_v11b.py run \
  --output e2e_output/project_001 \
  --stop-after-stage char_refs

断点恢复：已有的 storyboard.json 不会重新生成。

5 Agent 审查角色图，抽卡
# 查看角色图候选
python scripts/e2e_v11b.py review char_refs \
  --output e2e_output/project_001 --json

# 对男主角抽两次卡
python scripts/e2e_v11b.py reroll char_ref \
  --output e2e_output/project_001 --char char_002
python scripts/e2e_v11b.py reroll char_ref \
  --output e2e_output/project_001 --char char_002

# 查看 3 个候选
python scripts/e2e_v11b.py list-candidates char_ref \
  --output e2e_output/project_001 --char char_002 --json

# Agent 选择第 2 个（最满意的）
python scripts/e2e_v11b.py select char_ref \
  --output e2e_output/project_001 --char char_002 --candidate 2
6 继续执行到视频生成完毕
python scripts/e2e_v11b.py run \
  --output e2e_output/project_001 \
  --stop-after-stage video_gen
7 Agent 审查视频，抽卡不满意的子镜头
# 查看所有视频
python scripts/e2e_v11b.py review videos \
  --output e2e_output/project_001 --json

# Seg3-Sub2 视频效果不好，重新抽卡
python scripts/e2e_v11b.py reroll video \
  --output e2e_output/project_001 --seg 3 --sub 2

# 选择新生成的版本
python scripts/e2e_v11b.py select video \
  --output e2e_output/project_001 --seg 3 --sub 2 --candidate 2
8 跑完组装 + 质量检测
python scripts/e2e_v11b.py run --output e2e_output/project_001

自动跳过已完成的 Stage 1-7，只执行 Stage 8（组装）和 Stage 9（质量检测）。

7. CLI 命令参考

run — 执行流程

python scripts/e2e_v11b.py run [OPTIONS]
  --output DIR              输出目录（必需）
  --duration INT            目标时长，秒（默认 40）
  --voice STR               TTS 音色（默认 female-shaonv）
  --input FILE              输入文本文件（默认 data/test_novel.txt）
  --stop-after-stage STR    在指定 stage 后暂停
  --stop-after-segment INT  在指定段号后暂停（仅 video_gen stage）

review — 审查状态

python scripts/e2e_v11b.py review TARGET [OPTIONS]
  TARGET: storyboard | tts | char_refs | scene_bgs | first_frames | videos | status
  --output DIR     输出目录
  --json           JSON 格式输出

edit — 编辑分镜

python scripts/e2e_v11b.py edit storyboard [OPTIONS]
  --output DIR     输出目录
  --segment INT    段号（必需）
  --field STR      字段名（必需）：narration_text, emotion, image_prompt, video_prompt, ...
  --value STR      新值（必需）
  --sub INT        子镜头号（1-based，仅编辑 sub_shot 字段时）

reroll — 抽卡

python scripts/e2e_v11b.py reroll TARGET [OPTIONS]
  TARGET: char_ref | scene_bg | tts | first_frame | video
  --output DIR     输出目录
  --char STR       角色 ID（char_ref 时必需）
  --scene STR      场景 ID（scene_bg 时必需）
  --seg INT        段号（tts/video 时必需）
  --sub INT        子镜头号（video 时必需，1-based）
  --voice STR      TTS 音色覆盖
  --emotion STR    TTS 情感覆盖

select — 选择候选

python scripts/e2e_v11b.py select TARGET [OPTIONS]
  TARGET: char_ref | scene_bg | tts | first_frame | video
  --output DIR        输出目录
  --candidate INT     候选版本号（必需）
  --char / --scene / --seg / --sub   资产标识

list-candidates — 列出候选项

python scripts/e2e_v11b.py list-candidates TARGET [OPTIONS]
  TARGET: char_ref | scene_bg | tts | first_frame | video
  --output DIR     输出目录
  --json           JSON 输出

8. 编辑失效级联表

编辑内容自动失效的下游资产
narration_text（段 N）该段 TTS + 该段所有子镜头视频
emotion（段 N）该段 TTS
image_prompt（段 N）该段首帧图
video_prompt（段 N 子镜头 M）对应子镜头视频
appearance_prompt（角色 X）角色参考图 + 所有引用该角色的视频
选择新角色参考图所有引用该角色的视频
选择新首帧图对应子镜头视频
失效的资产在下次执行 run 时自动重新生成，不需要手动处理。

9. FAQ

Q: 抽卡（reroll）会覆盖之前的结果吗？

不会。每次 reroll 生成新的候选项（v1, v2, v3...），旧的保留在磁盘上。通过 select 命令选择要使用的版本。

Q: 下载失败会浪费额度吗？

不会。视频生成消耗额度发生在提交任务时。下载失败后，可以用同一个 task_id 重新获取 URL 下载，不额外消耗。系统内置 3 次下载重试。

Q: 编辑分镜后需要重跑全部吗？

不需要。系统会自动标记受影响的下游资产为"失效"，下次 run 时只重新生成失效的部分，其他已完成的资产保持不变。

Q: Agent 怎么知道选哪个候选项？

Agent 调用 list-candidates --json 获取所有候选项的文件路径，然后用视觉能力（如 Qwen 3.5-Plus 视频/图片理解）评估质量，选择最佳的。

Q: 支持哪些 TTS 情感？

happy, sad, angry, fearful, surprised, calm, disgusted, fluent。
whisper 会自动映射为 calm（MiniMax 不支持 whisper）。

Q: 视频时长为什么是 3-5 秒而不是固定 5 秒？

TTS 先生成，真实时长驱动子镜头时长分配。如果旁白较短（TTS=5s），两个子镜头会被缩短为 3s+3s 而非浪费 5s+5s。节省约 30% 视频生成费用。
